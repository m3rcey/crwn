*Pull back to show the full whiteboard: all 11 names organized under 3 departments. Hold for 3 seconds.*

## HOW IT WORKS (60 seconds)

This isn't a gimmick. Let me show you how it actually works.

*Screen recording: terminal showing the .claude/agents/ directory tree — the same view as the folder structure on the whiteboard.*

Three of these agents run on a schedule in the cloud. Amara runs every morning. Devon runs every Monday. Zara runs every Monday. They check in, do their job, and report back. I don't touch anything.

*Screen recording: Claude Code scheduled agents dashboard showing the 3 triggers with their schedules and next run times.*

But the real power is what happens inside the platform every day. At 1 PM, the AI manager runs for every artist on the platform. It collects their data — revenue, subscribers, churn, plays, community engagement. It generates insights. And then it acts. Low-risk actions get executed automatically. High-risk actions get escalated for the artist to approve.

*Screen recording: AI Manager tab showing pending actions with Approve/Reject buttons, recent agent activity with green checkmarks.*

When an action executes, the system takes a snapshot of the artist's metrics. Seven days later, it measures again. Did MRR go up? Did churn go down? Did subscribers increase? That delta gets stored. And the next time the AI runs, it sees what worked and what didn't. It learns.

*Whiteboard — draw a simple loop: "Action" -> "Snapshot" -> "7 days" -> "Measure" -> "Learn" -> back to "Action". Write "FEEDBACK LOOP" above it.*

And then Orion takes those results from every artist and feeds them back to every other artist. One artist's win becomes every artist's advantage. The more artists on the platform, the smarter every agent gets.

*Tap the loop on the whiteboard. Circle "FEEDBACK LOOP" for emphasis.*

Every artist who joins brings fans. Every fan generates data. Every data point makes the agents smarter. Every agent improvement helps every artist. It's a flywheel. And it's already spinning.
